# 发布后工作流

内容发布后的数据收集、分析和优化建议流程。

## 数据收集

### 自动收集指标

| 指标 | 说明 | 数据来源 |
|-----|------|---------|
| viewCount | 阅读量/播放量 | 平台 API |
| likeCount | 点赞数 | 平台 API |
| commentCount | 评论数 | 平台 API |
| shareCount | 转发数 | 平台 API |
| collectCount | 收藏数 | 平台 API |
| avgReadTime | 平均阅读时长 | 平台 API |
| engagementRate | 互动率 | 计算得出 |

### 平台支持

#### 微信公众号
- 通过公众号后台 API 获取文章数据
- 定时同步（建议每小时一次）
- 包含用户画像数据

#### 小红书
- 数据需手动录入或浏览器插件采集
- 关键指标：笔记曝光、互动率、涨粉数

#### 今日头条
- 通过头条号后台 API 获取
- 包含推荐量、阅读完成率

#### 知乎
- 通过知乎专栏 API 获取
- 关键指标：赞同数、收藏数、评论数

## 数据分析

### 表现评估

#### 互动率计算
```
互动率 = (点赞 + 评论 + 转发 + 收藏) / 阅读量 × 100%
```

#### 表现等级
| 等级 | 互动率 | 建议动作 |
|-----|-------|---------|
| 优秀 | > 10% | 分析成功因素，复制到其他内容 |
| 良好 | 5-10% | 保持当前策略 |
| 一般 | 2-5% | 优化标题和开头 |
| 较差 | < 2% | 考虑修改或重新创作 |

### 高质量内容识别

#### 特征分析
系统会分析表现优秀的文章特征：
- 标题长度分布
- 开头风格偏好
- 内容结构特点
- 发布时间规律

#### 成功因素提取
- 选题角度是否独特
- 素材使用是否有效
- 风格是否匹配受众
- 发布时机是否合适

## 反馈循环

### 数据驱动优化

#### 标题优化
- 分析高点击率标题特征
- 生成 A/B 测试建议
- 提供优化方向

#### 开头优化
- 分析高停留率开头特征
- Hook 效果评估
- 改进建议

#### 内容优化
- 阅读完成率分析
- 用户流失点识别
- 内容调整建议

### 自动建议生成

#### 低表现内容
```
检测到「文章标题」表现低于平均水平：
- 互动率：2.1%（平台平均：5.3%）
- 建议：优化开头 hook，增加互动引导
```

#### 高表现内容
```
检测到「文章标题」表现优秀：
- 互动率：12.5%
- 成功因素：故事开头 + 情感共鸣
- 建议：可考虑改编为其他平台内容
```

## 跨平台适配建议

### 数据驱动的平台适配

基于目标平台的数据特征，系统会提供：
- 标题长度建议
- 开头风格建议
- 内容结构建议
- 发布时间建议

### 平台特征对照

| 平台 | 最佳标题长度 | 开头风格 | 最佳发布时间 |
|-----|------------|---------|------------|
| 微信公众号 | 20-30字 | 悬念/故事 | 早8点、晚9点 |
| 小红书 | 15-20字 | 直接/实用 | 晚7-10点 |
| 今日头条 | 25-35字 | 热点/数据 | 早7点、午12点 |
| 知乎 | 20-40字 | 问题/观点 | 晚8-11点 |

## 报告生成

### 周度报告
- 本周发布内容数量
- 总阅读量/互动量
- 表现最佳内容 TOP 3
- 表现最差内容及建议

### 月度报告
- 内容增长趋势
- 互动率变化趋势
- 高质量内容特征总结
- 下月优化方向建议

## 数据存储

### 存储策略
- 原始数据：永久保存
- 聚合数据：按周/月存储
- 分析结果：关联到文章

### 数据安全
- 敏感数据加密存储
- 访问权限控制
- 定期备份

## API 接口

### 获取文章数据
```
GET /api/article/:id/metrics
```

### 获取表现最佳内容
```
GET /api/article/top-performing?limit=10&platform=wechat
```

### 手动录入数据
```
POST /api/article/:id/metrics
{
  "viewCount": 10000,
  "likeCount": 500,
  ...
}
```

## 最佳实践

### DO ✅
- 定期查看数据报告
- 分析高表现内容特征
- 根据数据调整策略
- 保持数据录入的及时性

### DON'T ❌
- 只看阅读量忽略互动率
- 忽视低表现内容
- 数据孤岛，不与其他内容对比
- 过度依赖数据，忽略内容质量
