# 内容修改工作流

当需要对已发布或草稿状态的文章进行修改时的工作流程。

## 触发条件

1. **用户主动要求修改**：收到修改指令
2. **HKR 评分过低**：综合评分低于 6 分
3. **质量检查未通过**：AI味过重、逻辑漏洞等
4. **发布后反馈**：收到用户反馈需要修改
5. **数据表现不佳**：阅读量、互动率低于预期

## 修改类型

| 类型 | 描述 | 处理方式 |
|-----|------|---------|
| 小修 | 错别字、标点、格式 | 直接修改，无需审校 |
| 中修 | 段落调整、内容补充 | 修改后进行一遍审校 |
| 大修 | 结构调整、重写部分 | 按新创作流程执行 |

## 修改工作流

### Step 1: 分析修改需求

#### 明确修改范围
- 具体要修改哪些部分？
- 修改的目标是什么？
- 是否影响整体结构？

#### 评估修改影响
- 修改后是否需要更新大纲？
- 是否需要补充新的素材？
- 是否需要重新进行 HKR 评估？

### Step 2: 定位问题

#### 自动检测问题
- AI 味关键词扫描
- 逻辑漏洞检测
- 事实核查提醒

#### 人工确认
- 与原始需求对照
- 确认修改优先级

### Step 3: 执行修改

#### 小修改流程
```
定位问题 → 直接修改 → 保存版本
```

#### 中修改流程
```
定位问题 → AI 辅助修改 → 一遍审校 → 保存版本
```

#### 大修改流程
```
分析需求 → 回到创作工作流相应步骤 → 重新生成 → 三遍审校 → 保存版本
```

### Step 4: 版本管理

- 每次修改自动创建新版本
- 保留最近 10 个版本
- 支持版本对比和回滚

## 与新创作流程的衔接

### 从哪个步骤开始？

| 修改类型 | 建议起始步骤 | 原因 |
|---------|------------|------|
| 仅修改措辞 | Step 8（审校） | 不影响整体结构 |
| 补充内容 | Step 5（素材库） | 需要新素材支撑 |
| 调整角度 | Step 1（选题讨论） | 需要重新确定方向 |
| 重写部分 | Step 7（初稿生成） | 需要重新生成内容 |
| 整体重写 | Step 0（理解需求） | 按新创作流程执行 |

### 保留已有成果

修改时系统会：
- ✅ 保留原有选题分析（除非明确要求更换）
- ✅ 保留已有素材（可以追加新素材）
- ✅ 保留风格设置（除非明确要求更换）
- ✅ 记录修改历史

## 修改指令示例

### 小修改
```
把第三段的"显著提升"改成具体数字
修正第二段的数据引用错误
调整标题长度到 30 字以内
```

### 中修改
```
补充一个真实案例到第四部分
把开头的引言改成故事开头
增加一个小结段落
```

### 大修改
```
把观点角度从"技术分析"改成"用户体验"
重写后半部分，补充最新的行业数据
整篇文章改写成小红书风格
```

## 质量保证

### 修改后检查清单
- [ ] 修改目标是否达成
- [ ] 是否引入新的 AI 味
- [ ] 逻辑是否仍然连贯
- [ ] 数据是否仍然准确
- [ ] 风格是否仍然统一

### HKR 重评
大修改后建议重新进行 HKR 评估，确保：
- H 分数未降低
- K 分数有所提升
- R 分数保持稳定

## 异常处理

### 修改后质量下降
1. 回滚到上一版本
2. 分析下降原因
3. 调整修改策略

### 反复修改仍不满意
1. 考虑重新创作
2. 收集更多素材
3. 尝试不同风格模板

### 修改意见冲突
1. 保留原始版本
2. 创建多个修改版本
3. 人工决策最终版本
