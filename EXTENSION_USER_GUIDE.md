## 使用流程

### 第一步：配置平台

1. 打开 **发布管理 - 平台配置** 页面
2. 为需要使用的平台配置以下信息：
   - 平台名称（如：微博、B站等）
   - 发布方式：选择 **"扩展"** 或 **"混合"**
   - 扩展配置（可选）

### 第二步：准备内容

在系统中创建要发布的内容：
- **文章**：在文章管理页面创建
- **视频**：在项目管理页面创建
- **图片**：在资源管理页面上传

### 第三步：创建发布任务

有两种方式创建发布任务：

#### 方式1：智能发布（推荐）
```
POST /api/publish/smart
{
  "contentType": "article",
  "contentId": "文章ID",
  "platformId": "平台ID",
  "mode": "publish"  // 或 "draft"
}
```

系统会自动选择最佳发布方式（API 或扩展）。

#### 方式2：批量发布
```
POST /api/publish/batch
{
  "contentType": "article",
  "contentIds": ["id1", "id2"],
  "platformIds": ["platform1", "platform2"],
  "mode": "publish"
}
```

可以同时发布多个内容到多个平台。

### 第四步：扩展自动执行

1. 扩展每 5 秒轮询一次后端，获取待处理任务
2. 扩展自动打开对应平台页面
3. 扩展自动填充内容并发布
4. 发布完成后，扩展将结果回传给后端

### 第五步：查看发布记录

在 **发布管理 - 发布记录** 页面可以查看：
- 发布状态（待处理、处理中、成功、失败）
- 发布方式（API 或扩展）
- 扩展执行日志
- 发布时间和结果

## 工作原理

### 通信机制

```
前端页面 ──────────────────────────────────────┐
    │                                          │
    │ 1. 创建发布任务                           │
    ↓                                          │
后端服务器                                      │
    │                                          │
    │ 2. 任务存入队列                           │
    │                                          │
    │ 3. 扩展轮询获取任务 ←─────────────────────┤
    ↓                                          │
浏览器扩展                                      │
    │                                          │
    │ 4. 打开平台页面                           │
    │ 5. 自动填充内容                           │
    │ 6. 执行发布操作                           │
    │                                          │
    │ 7. 回传执行结果 ──────────────────────────┘
    ↓
后端服务器
    │
    │ 8. 更新发布记录
    ↓
前端页面显示结果
```

### 心跳机制

- 扩展每 30 秒发送一次心跳到后端
- 后端判断最后心跳时间在 60 秒内则认为在线
- 前端每 30 秒检测一次扩展状态

## 高级功能

### 1. 定时发布

```
POST /api/publish/scheduled
{
  "contentType": "article",
  "contentId": "文章ID",
  "platformId": "平台ID",
  "mode": "publish",
  "scheduledAt": "2026-02-09T10:00:00Z"
}
```

系统会在指定时间自动执行发布。

### 2. 批量发布进度追踪

创建批量发布后，可以实时查看进度：

```
GET /api/publish/batch/{batchId}/progress
```

返回：
- 总任务数
- 待处理数
- 成功数
- 失败数
- 详细记录列表

### 3. 取消批量发布

```
POST /api/publish/batch/{batchId}/cancel
```

## 注意事项

### 1. 浏览器要求
- 必须保持浏览器打开
- 扩展必须处于启用状态
- 建议使用 Chrome 或 Edge 浏览器

### 2. 平台登录
- 发布前需要在浏览器中登录对应平台
- 扩展会复用浏览器的登录状态
- 无需在系统中配置平台账号密码

### 3. 发布限制
- 遵守各平台的发布频率限制
- 避免短时间内大量发布
- 建议使用批量发布的并发控制

### 4. 错误处理
- 发布失败会自动重试（最多 3 次）
- 可以在发布记录中查看详细错误日志
- 支持手动重新发布失败的任务

## 故障排查

### 扩展显示离线
1. 检查扩展是否启用
2. 检查后端服务是否运行
3. 查看 Service Worker 控制台日志
4. 刷新扩展或重新加载

### 发布任务一直待处理
1. 检查扩展是否在线
2. 查看 Service Worker 控制台是否有错误
3. 检查平台是否已登录
4. 查看任务队列是否有积压

### 发布失败
1. 查看发布记录中的错误日志
2. 检查平台页面结构是否变化
3. 确认内容格式是否符合平台要求
4. 手动在平台上测试发布

## API 端点总览

### 扩展管理
- `GET /api/extension/status` - 检测扩展状态
- `POST /api/extension/ping` - 扩展心跳
- `GET /api/extension/tasks/pending` - 获取待处理任务
- `POST /api/extension/tasks/:id/status` - 更新任务状态
- `POST /api/extension/tasks/:id/complete` - 完成任务
- `POST /api/extension/tasks/:id/fail` - 任务失败

### 智能发布
- `POST /api/publish/smart` - 智能发布
- `GET /api/publish/platforms/:id/capabilities` - 获取平台能力

### 批量发布
- `POST /api/publish/batch` - 创建批量发布
- `GET /api/publish/batch/:id/progress` - 获取进度
- `POST /api/publish/batch/:id/cancel` - 取消发布

### 定时发布
- `POST /api/publish/scheduled` - 创建定时发布
